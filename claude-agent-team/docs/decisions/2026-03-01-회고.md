## 배운 패턴 / 교훈

1. **"기능 먼저, 보안은 나중"은 기술 부채가 아니라 실제 위협**: RCE 수준의 취약점이 발견됨. 다음 프로젝트에서는 첫 API 라우트 작성 시점부터 미들웨어 인증 추가.

2. **순수 함수 설계의 가치**: `agent-router.ts`를 side-effect 없는 순수 함수로 구현했기 때문에 smoke test 10개 작성이 쉬웠고, 리뷰어가 로직을 빠르게 검증할 수 있었다.

3. **plan.md 지속 업데이트 효과**: 매 세션 후 `docs/plan.md`를 업데이트한 덕분에 다음 세션에서 컨텍스트 재구축 시간이 최소화됨. 새 에이전트(reviewer 등)도 파일 1개로 전체 상태 파악 가능.

4. **에이전트 memory 파일 = 품질 보증 필요**: 현재 developer.md는 툴 실행 로그가 그대로 저장되어 있어 유용성이 낮다. 메모리는 구조화된 Key-Value 또는 섹션 형식으로만 저장해야 한다.

5. **Windows 환경 특수성**: `npx`, `.cmd`, `&` 같은 Unix/Windows 차이가 반복적으로 문제를 일으킴. `start-server.js` 래퍼 패턴은 효과적이었으며, 향후 프로젝트에서도 재사용 가능.

6. **Shadow 모드의 안전한 실험 방법**: 실제 에이전트 호출 없이 다음 홉 후보만 예측하는 Shadow 모드로 Multi-Hop 라우팅을 사전 검증 후 Phase 2에서 실제 실행으로 전환하는 점진적 접근 방식은 좋은 패턴.
